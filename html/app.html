Transaction Director Autoscaling

Reconnecting...

This DBOS workflow periodically checks the health of Transaction Director's clients, decommissioning the unhealthy ones and provisioning replacements. It runs every minute on a schedule, and can also be triggered manually from the button below.

DBOS makes the loop resilient to any failure: workflow state is durably persisted to Postgres, so if the process crashes mid-run it resumes from the last completed step on the next launch.

Try it: launch a run, then crash the application while it's mid-flight. When the app restarts, the workflow recovers and finishes the remaining client replacements. 

// Sub-workflow: returns the unhealthy clientsfunc checkUnhealthyClients(ctx, clients) {  for client := range clients {    isCongested := dbos.RunAsStep(ctx, getCongestionSignal)    if isCongested { unhealthy = append(unhealthy, client) }  }  return unhealthy}
 
// Sub-workflow: replace one unhealthy clientfunc replaceClient(ctx, client) {  dbos.RunAsStep(ctx, decommissionClient)  machine    := dbos.RunAsStep(ctx, startNewMachine)  newClient  := dbos.RunAsStep(ctx, runStartupScript)  dbos.RunAsStep(ctx, registerNewClient)}
 
// Main scheduled workflow: runs every minutefunc autoscalingWorkflow(ctx, scheduledTime) {  unhealthy := dbos.RunWorkflow(ctx, checkUnhealthyClients)  for client := range unhealthy {    dbos.RunWorkflow(ctx, replaceClient, client)  }}dbos.RegisterWorkflow(autoscalingWorkflow,    dbos.WithSchedule("0 * * * * *")) // every minute

The workflow is scheduled to run every minute via dbos.WithSchedule("0 * * * * *"). Manually launching it from the button above triggers an extra ad-hoc run.

While a run is in flight, hit "Crash the application." When the process restarts you'll see (Recovered pending workflows...) in the logs and the loop continues from the last completed step.
